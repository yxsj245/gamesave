# GameSave Manager - 使用文档

> 一款简洁的 Windows 桌面游戏存档备份与恢复工具，基于 WinUI 3 构建。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| 游戏管理 | 添加、编辑、删除游戏，管理游戏存档目录 |
| 手动备份 | 随时手动为游戏存档创建备份点 |
| 自动备份 | 游戏退出后自动备份存档（需配置启动进程） |
| 恢复存档 | 从备份点恢复游戏存档 |
| 启动游戏 | 通过软件启动游戏并自动管理存档生命周期 |
| 系统托盘 | 关闭窗口时最小化到托盘，后台常驻运行 |
| 设置 | 自定义工作目录、外观主题切换 |

---

## 快速开始

### 1. 添加游戏

1. 在主页点击工具栏上的 **「添加游戏」** 按钮
2. 在弹窗中填写以下信息：
   - **游戏名称（必填）**：为游戏起一个易于辨认的名称
   - **游戏存档目录（必填）**：选择游戏存档文件所在的目录
   - **游戏启动进程（可选）**：选择游戏的可执行文件（`.exe`），用于启动游戏和监测运行状态
   - **启动附加参数（可选）**：游戏启动时的命令行参数
   - **云端服务商（可选）**：选择已配置的云端服务商进行存档同步
3. 点击 **确定** 完成添加

> 💡 如果存档目录中已有文件，软件会自动创建一份初始备份（标记为「退出存档」）。

> 🎮 **自动图标**：如果配置了游戏启动进程（`.exe`），游戏列表中将自动提取并显示该 EXE 的图标。图标会缓存到本地，编辑游戏更换启动进程后图标会自动刷新。

> 🔄 **多设备支持**：选择存档目录后，路径会自动替换为 Windows 环境变量形式（如 `%APPDATA%\Game\Saves`），以便在不同设备间导入导出时路径能自动适配。支持的环境变量包括：`%LOCALAPPDATA%`、`%APPDATA%`、`%USERPROFILE%`、`%PUBLIC%`、`%PROGRAMFILES%`、`%PROGRAMFILES(X86)%`、`%PROGRAMDATA%`、`%SYSTEMDRIVE%`。

### 1.1 编辑游戏属性

1. 在主页的游戏列表中，**右键点击**要编辑的游戏卡片
2. 在弹出的右键菜单中选择 **「编辑游戏」**
3. 在弹窗中可修改以下信息：
   - **游戏名称**：修改游戏的显示名称
   - **游戏存档目录**：展示当前存档目录路径（**不可修改**）
   - **游戏启动进程**：修改或添加游戏启动程序路径
   - **启动附加参数**：修改启动参数
   - **云端服务商**：更换或取消关联的云端配置
4. 点击 **保存** 应用修改

> ⚠️ 游戏存档目录在添加后不可更改，这是为了避免存档目录变更导致已有备份失效。

### 2. 查看存档

- 在主页点击任意游戏卡片，弹出该游戏的详情面板
- 面板中展示了该游戏的所有存档列表，包括：
  - **退出存档**：游戏退出后自动创建的备份，仅保留最新一份，带有 <span style="color:red">自动</span> 标签
  - **手动存档**：用户手动创建的备份，可有多份

### 3. 手动备份

1. 打开游戏详情面板
2. 点击顶部的 **「备份」** 按钮
3. 可输入自定义存档名称（留空则默认为「手动存档」）
4. 点击 **备份** 完成

### 4. 恢复存档

1. 在游戏详情面板中找到要恢复的存档
2. 点击该存档右侧的 **「还原」** 按钮
3. 软件将自动清空游戏存档目录并恢复备份内容

> ⚠️ **注意**：
> - 仅「手动存档」支持恢复操作
> - 如果检测到游戏正在运行，会弹出二次确认警告

### 5. 删除存档

- 在游戏详情面板中，点击手动存档右侧的 **删除按钮**
- 在确认弹窗中可选择是否 **同时删除云端存档**（仅当游戏关联了云端配置时显示此选项）
- 确认后即可删除该存档

> 「退出存档」由系统自动管理，不支持手动删除。
>
> 💡 勾选「同时删除云端存档」后，软件会根据本地存档文件名匹配云端对应的存档并删除。云端删除失败不会影响本地删除。

### 5.1 批量删除存档

1. 在游戏详情面板中，点击存档列表右上角的 **「批量删除」** 按钮
2. 存档列表变为多选模式，勾选要删除的存档
3. 顶部按钮会实时显示已选数量（如「删除所选 (3)」）
4. 点击 **「删除所选」** 按钮，在确认弹窗中可选择是否 **同时删除云端存档**
5. 确认后批量删除
6. 点击 **「取消」** 可退出批量删除模式

> 退出存档会被自动跳过，不会被删除。

### 5.2 批量删除云端存档

1. 进入 **云端** 页面，选择云端配置
2. 点击顶部工具栏的 **「批量删除」** 按钮
3. 存档列表中每个存档前方会出现复选框，勾选要删除的存档
4. 顶部按钮会实时显示已选数量
5. 点击 **「删除所选」** 按钮，确认后批量删除
6. 点击 **「取消」** 可退出批量删除模式
7. 删除完成后会自动刷新云端存档列表

### 6. 启动游戏

1. 打开游戏详情面板
2. 点击顶部的 **「启动」** 按钮
3. 软件将自动执行以下流程：
   - 恢复最新的退出存档到游戏存档目录
   - 启动游戏进程
   - 后台监测游戏进程
   - 检测到游戏退出后，自动备份当前存档

> 需要预先在添加游戏时设置「游戏启动进程」。

## 设置

点击左侧导航栏底部的 **⚙️ 设置** 即可进入设置页面。

### 外观

选择应用的显示主题，支持以下三种模式：

| 模式 | 说明 |
|------|------|
| 跟随系统 | 自动跟随 Windows 系统的深色/浅色设置（默认） |
| 浅色模式 | 始终使用浅色主题 |
| 深色模式 | 始终使用深色主题 |

> 切换后立即生效，无需重启应用。

### 工作目录

所有游戏存档备份文件保存在工作目录下。

- **默认路径**：`C:\Users\<用户名>\xingchen\ycyaw\.gamesave`
- 可通过 **「更改」** 按钮选择新的工作目录

## 存档标签说明

| 标签 | 创建方式 | 数量 | 可恢复 | 可删除 |
|------|----------|------|--------|--------|
| 🔴 退出存档 | 游戏退出后自动创建 | 仅保留最新 1 份 | ❌ | ❌ |
| 🟢 手动存档 | 用户手动点击备份 | 无限制 | ✅ | ✅ |

## 文件名格式

存档备份文件为 `.tar` 归档格式，文件名格式为：

```
{Unix时间戳}_{存档标签}.tar
```

例如：
- `1709234567_退出存档.tar`
- `1709234890_手动存档.tar`
- `1709235123_打败了Boss.tar`

## 工作目录结构

```
.gamesave/
├── <游戏ID-1>/                    # 以游戏唯一标识命名
│   ├── 1709234567_退出存档.tar    # 退出时自动备份
│   ├── 1709234890_手动存档.tar    # 用户手动备份
│   └── 1709235123_手动存档.tar
├── <游戏ID-2>/
│   └── ...
└── config.json                    # 全局配置文件
```

## 常见问题

**Q: 恢复存档后游戏存档目录会怎样？**  
A: 恢复操作会先清空游戏存档目录中的所有内容，然后将备份的 `.tar` 文件解压到该目录。

**Q: 退出存档为什么不能手动恢复和删除？**  
A: 退出存档由系统自动管理，作为启动游戏时的自动恢复点。始终保留最新一份以确保游戏数据安全。

**Q: 工作目录可以放在移动硬盘上吗？**  
A: 可以，但请确保每次使用前移动硬盘已连接。

**Q: 关闭窗口后程序退出了吗？**  
A: 没有。点击窗口关闭按钮后，程序会最小化到系统托盘（任务栏右下角通知区域）继续在后台运行。首次最小化时会弹出气泡提示。

**Q: 如何完全退出程序？**  
A: 在系统托盘找到 GameSave Manager 的图标，右键点击选择 **「退出」** 即可完全关闭程序。

**Q: 如何从托盘恢复主窗口？**  
A: 双击系统托盘中的 GameSave Manager 图标，或右键点击选择 **「显示主窗口」**。
