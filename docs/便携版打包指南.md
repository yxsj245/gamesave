# 便携版打包指南

## 概述

便携版（Portable）是一种免安装的分发方式，生成一个单独的 EXE 文件，双击即可运行。

**工作原理**：脚本将所有应用文件打包为 ZIP 并嵌入一个自解压启动器 EXE 中。首次运行时自动解压程序到 `%LOCALAPPDATA%\GameSave-Portable`（用户无感），工作目录（配置和存档数据）自动创建在 **EXE 旁边的 `GameSave-Data` 文件夹**中。

这样只需拷贝便携版 EXE 和 `GameSave-Data` 文件夹即可实现数据迁移。

## 环境要求

| 要求 | 版本 |
|------|------|
| Windows | 10 (1809+) / 11 |
| .NET SDK | 10.0+ |
| .NET Framework | 4.x（系统自带，用于编译启动器） |

## 构建方法

### 基本用法

```powershell
# 默认构建 x64 便携版
.\build-portable.ps1

# 指定平台和版本号
.\build-portable.ps1 -Platform x64 -Version 1.2.0

# 跳过发布步骤（使用已有的发布文件）
.\build-portable.ps1 -SkipPublish
```

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `-Platform` | `x64` | 目标平台：`x64`、`x86`、`ARM64` |
| `-Version` | `1.0.0` | 版本号，格式 `Major.Minor.Build` |
| `-SkipPublish` | 无 | 跳过 dotnet publish 步骤 |

### 构建输出

```
installer-output/
└── GameSave-x64-Portable-v1.0.0.exe    # 单文件自解压 EXE
```

## 用户使用方法

1. 双击 `GameSave-x64-Portable-vX.X.X.exe`
2. 程序自动解压到系统目录（用户无感）
3. 工作目录自动创建在 EXE 旁边的 `GameSave-Data` 文件夹
4. 配置和存档数据都保存在 `GameSave-Data` 中
5. 后续运行直接启动，无需再次解压
6. 版本更新时下载新的 EXE 替换即可，数据会保留

> **注意**：自包含（SelfContained）部署，无需预装 .NET 运行时。

## 便携版目录结构

```
某个文件夹/（如 U 盘）
├── GameSave-x64-Portable-v1.0.0.exe    # 便携版启动器
└── GameSave-Data/                       # 工作目录（只有用户数据）
    ├── config.json                      # 配置文件
    └── ...                              # 游戏存档备份等数据
```

程序运行文件解压在：`%LOCALAPPDATA%\GameSave-Portable`（对用户不可见）

## 便携模式检测原理

启动器通过 `--portable-workdir` 命令行参数将工作目录路径传递给应用：

```
GameSave.exe --portable-workdir "C:\便携版\GameSave-Data"
```

`ConfigService` 解析此参数：
- **参数存在**：进入便携模式，使用参数指定的路径作为工作目录
- **参数不存在**：使用普通安装模式的默认路径（`%USERPROFILE%\xingchen\ycyaw\.gamesave`）

## 与 MSI 安装版的区别

| 特性 | 便携版 | MSI 安装版 |
|------|--------|-----------| 
| 安装方式 | 双击 EXE 直接运行 | 需要安装 |
| 数据存储位置 | EXE 旁边的 `GameSave-Data` | 用户目录 |
| 程序文件位置 | `%LOCALAPPDATA%\GameSave-Portable` | 安装目录 |
| 开始菜单快捷方式 | ❌ | ✅ |
| 桌面快捷方式 | ❌ | ✅ |
| 卸载方式 | 删除 EXE + `GameSave-Data` + `%LOCALAPPDATA%\GameSave-Portable` | 控制面板卸载 |
| 适用场景 | U 盘携带、多设备同步 | 长期使用 |

## 技术细节

构建过程分为三步：
1. **发布应用**：使用 `dotnet publish` 发布 SelfContained + SingleFile 应用
2. **打包 ZIP**：将所有发布文件（排除 .pdb）打包为 ZIP
3. **生成自解压 EXE**：使用 csc 编译 C# 启动器，将 ZIP 追加在 EXE 末尾

EXE 文件格式：`[启动器 EXE][ZIP 数据][8字节偏移量]`
