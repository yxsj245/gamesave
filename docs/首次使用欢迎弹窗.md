# 首次使用欢迎弹窗

## 功能说明

当用户首次打开 GSAM 游戏存档管理器时，会自动弹出一个欢迎弹窗（`ContentDialog`），展示公测须知和联系方式。弹窗只会在首次使用时显示一次，后续启动不再弹出。

## 弹窗内容

弹窗包含以下信息：

1. **标题**：欢迎使用 GSAM 游戏存档管理器（公测）
2. **公测须知**：说明当前为公测版本，功能和界面不代表最终效果
3. **交流与反馈**：
   - QQ：3354416548
   - QQ群：1053482216
4. **底部感谢语**

## 技术实现

### 配置持久化

在 `AppConfig` 中新增 `HasShownWelcome` 布尔字段（默认 `false`），用于标记是否已显示过欢迎弹窗。该字段会随 `config.json` 一起保存和读取。

- **配置路径**：`Services/ConfigService.cs` → `AppConfig.HasShownWelcome`
- **设置方法**：`ConfigService.SetHasShownWelcomeAsync()` —— 将标记设为 `true` 并持久化

### 弹窗显示逻辑

在 `MainPage.xaml.cs` 的 `Page_Loaded` 事件中：

1. 页面加载后先导航到首页
2. 调用 `ShowWelcomeDialogIfFirstLaunchAsync()` 方法
3. 检查 `App.ConfigService.HasShownWelcome`：
   - 如果为 `true`（已显示过），直接返回
   - 如果为 `false`（首次使用），显示 `ContentDialog` 弹窗
4. 用户点击"我知道了"关闭弹窗后，调用 `SetHasShownWelcomeAsync()` 保存标记

### 涉及文件

| 文件 | 修改内容 |
|------|---------|
| `Services/ConfigService.cs` | `AppConfig` 添加 `HasShownWelcome` 属性；`ConfigService` 添加属性和 `SetHasShownWelcomeAsync()` 方法 |
| `Views/MainPage.xaml.cs` | `Page_Loaded` 改为 async；新增 `ShowWelcomeDialogIfFirstLaunchAsync()` 方法 |

## 如何重置（调试用）

如果需要重新触发欢迎弹窗（例如测试时），可以手动编辑 `config.json` 文件，将 `hasShownWelcome` 字段改为 `false`，或直接删除该字段。

配置文件路径：`%USERPROFILE%\xingchen\ycyaw\.gamesave\config.json`
