# 启动参数：工作目录设置

## 功能说明

通过 `--workdir` 启动参数指定程序的工作目录（存档备份文件的存储位置）。该参数具有**最高优先级**，生效后程序内的工作目录设置将变为灰色不可修改。

适用于 MSI 安装版和便携版。

## 使用方法

### 命令行启动

```bash
GameSave.exe --workdir "D:\MyGameSaves"
```

### 创建快捷方式

1. 右键 `GameSave.exe` → **发送到** → **桌面快捷方式**
2. 右键快捷方式 → **属性**
3. 在「目标」栏末尾追加参数：
   ```
   "C:\...\GameSave.exe" --workdir "D:\MyGameSaves"
   ```
4. 点击确定

### 便携版

便携版同样支持 `--workdir` 参数，启动器会自动将参数透传给内部应用程序：

```bash
GameSave-x64-Portable-v1.0.0.exe --workdir "D:\MyGameSaves"
```

## 参数优先级

从高到低：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1（最高） | `--workdir` | 命令行参数指定 |
| 2 | `--portable-workdir` | 便携版启动器自动传入 |
| 3 | `config.json` | 程序内设置页面更改 |
| 4（最低） | 默认目录 | 兜底值 |

## 行为说明

### 工作目录锁定

当 `--workdir` 参数生效时：
- 设置页面中的工作目录"更改"按钮变为**灰色不可点击**
- 显示提示文本：「工作目录由启动参数 --workdir 指定，无法在此更改」
- 工作目录仅在运行时生效，**不会写入** `config.json`

### 目录不存在处理

如果 `--workdir` 指定的目录不存在，程序启动时会：
1. 弹出错误对话框：「启动参数 --workdir 指定的工作目录不存在」
2. 用户点击「确定」后程序退出

### 开机自启动

开机自启动功能会自动保留 `--workdir` 参数。启用自启动后，注册表中的启动命令会包含 `--workdir` 参数，确保开机后使用相同的工作目录。
