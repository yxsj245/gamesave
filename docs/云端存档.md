# 云端存档功能 — 使用文档

> GameSave Manager 支持将游戏存档自动/手动同步到云端存储服务，当前已支持**阿里云 OSS**。

---

## 功能概述

- **自动上传**：游戏退出后备份完成时，自动将存档上传到已关联的云端存储
- **手动上传**：手动备份后，同样自动上传到关联的云端
- **游戏元数据同步**：上传存档时自动同步游戏配置信息（名称、存档路径、启动路径等）到云端
- **云端浏览**：在「云端」页面查看所有云端存档，按游戏分组展示
- **云端下载**：将云端存档下载到本地工作目录
- **云端删除**：删除不需要的云端存档
- **导入恢复**：本地没有对应游戏时，可一键从云端导入游戏配置并恢复存档

---

## 使用步骤

### 1. 配置阿里云 OSS

1. 进入 **设置** 页面
2. 在「云端存储」卡片中，点击 **添加配置**
3. 填写以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 配置名称 | 自定义显示名称 | `我的阿里云OSS` |
| Endpoint | OSS 端点地址 | `oss-cn-hangzhou.aliyuncs.com` |
| Bucket 名称 | 存储桶名称 | `my-game-saves` |
| AccessKey ID | 阿里云 AccessKey ID | `LTAI5t...` |
| AccessKey Secret | 阿里云 AccessKey Secret | `hBGr...` |
| 远端存储路径 | 存档在 OSS 中的根路径 | `GameSave`（默认值） |

4. 点击 **测试连接** 验证配置是否正确
5. 确认提示"连接成功"后，点击 **添加**

> ⚠️ **安全提示**：AccessKey 信息会以明文保存在本地配置文件 `config.json` 中，请妥善保管。建议使用 RAM 子账号的 AccessKey，仅授予 OSS 读写权限。

### 2. 关联游戏与云端配置

添加新游戏时：
1. 点击 **添加游戏**
2. 填写游戏信息后，在「云端服务商」下拉框中选择已配置的云端存储
3. 点击 **确定** 完成添加

> 已添加的游戏暂不支持修改云端关联，如需更改请删除后重新添加。

### 3. 自动同步

当游戏关联了云端配置后：
- 游戏退出后自动备份存档 → 自动上传到云端
- 手动备份存档 → 自动上传到云端

上传过程中，底部状态栏会显示 `☁️ 正在上传 xxx 存档到云端...`。

### 4. 查看云端存档

1. 点击左侧导航栏的 **云端** 页面
2. 在顶部下拉框中选择云端配置
3. 存档列表将按游戏分组展示，显示存档名称、备份时间和文件大小
4. 点击 **刷新** 按钮重新加载

### 5. 下载云端存档

在云端存档列表中，点击某个存档的 **下载** 按钮，存档将被下载到本地工作目录中对应游戏的子目录下。

### 6. 删除云端存档

在云端存档列表中，点击某个存档的 **删除** 按钮，确认后将从云端永久删除该存档。

## OSS 存储路径结构

云端存储的目录结构如下：

```
{RemoteBasePath}/
├── {游戏ID-1}/
│   ├── game.json                    ← 游戏元数据（自动同步）
│   ├── 1709234567_退出存档.tar
│   ├── 1709234890_手动存档.tar
│   └── ...
├── {游戏ID-2}/
│   ├── game.json
│   └── ...
```

- `game.json` 包含游戏参数（名称、存档路径、启动路径等），每次上传存档时自动更新
- 默认 `RemoteBasePath` 为 `GameSave`，即所有存档存储在 OSS Bucket 的 `GameSave/` 前缀下

## 阿里云 OSS 准备工作

如果您还没有阿里云 OSS，请按以下步骤准备：

1. 注册[阿里云账号](https://www.aliyun.com/)
2. 进入 [OSS 控制台](https://oss.console.aliyun.com/) 创建一个 Bucket
   - 地域建议选择离您最近的区域
   - 存储类型选择「标准存储」
   - 读写权限选择「私有」
3. 进入 [AccessKey 管理](https://ram.console.aliyun.com/manage/ak) 创建 AccessKey
   - 建议使用 RAM 子账号，仅授予 `AliyunOSSFullAccess` 权限
4. 记录下 Endpoint、Bucket 名称、AccessKey ID 和 AccessKey Secret

## 常见问题

### Q: 上传失败怎么办？
A: 请检查以下几点：
- 网络是否正常
- Endpoint 地址是否正确（注意不要加 `https://` 前缀）
- AccessKey 是否有效且有 OSS 读写权限
- Bucket 名称是否正确

### Q: 可以同时使用多个云端配置吗？
A: 可以。您可以添加多个云端配置，不同的游戏可以关联不同的云端配置。

### Q: 更换电脑后如何恢复？
A: 在新电脑上安装 GameSave Manager → 配置相同的 OSS 信息 → 进入云端页面 → 未导入的游戏会显示「导入恢复」按钮，点击即可一键导入游戏配置。也可以点击某个存档的「恢复到本地」按钮，一键完成导入+恢复。

### Q: 已有游戏如何关联云端？
A: 当前版本暂不支持已有游戏的云端关联修改，您需要删除游戏后重新添加并选择云端配置。后续版本将支持编辑功能。

### Q: 导入恢复时显示“缺少游戏信息，无法导入”怎么办？
A: 这说明该游戏的云端存档是在新版本之前上传的，未包含 `game.json` 元数据。解决方法：先手动添加游戏并关联云端配置，下次备份时会自动上传元数据。
