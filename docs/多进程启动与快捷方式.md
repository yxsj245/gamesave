# 多进程启动与快捷方式支持

## 功能概述

本功能增强了游戏启动进程管理能力，包括：

1. **快捷方式支持**：启动进程路径支持选择 `.lnk` 快捷方式文件，系统会自动解析快捷方式中的目标路径和启动参数
2. **多进程启动**：每个游戏最多支持配置两个启动进程（主进程 + 第二进程）
3. **顺序启动**：多进程模式下，按顺序先启动主进程，确认主进程成功运行后再启动第二进程
4. **崩溃检测**：任何进程在启动后 5 秒内退出将被视为启动失败/崩溃，自动终止监控周期
5. **组合退出**：多进程模式下，只有所有进程全部退出后才会触发存档备份

## 快捷方式解析

### 支持说明

- 在添加或编辑游戏的启动进程浏览对话框中，可直接选择 `.lnk` 快捷方式文件
- 选择后系统自动解析以下信息：
  - **目标路径**：快捷方式指向的实际 `.exe` 文件路径
  - **启动参数**：快捷方式中配置的命令行参数
- 解析成功后自动填充到路径和参数输入框中
- 如解析失败会弹窗提示用户手动选择 `.exe` 文件

### 技术实现

- 使用 Windows COM 接口 `IShellLinkW` 解析 `.lnk` 文件
- 实现类：`Helpers/ShortcutHelper.cs`
- 主要方法：
  - `IsShortcut(path)` - 判断是否为快捷方式
  - `ResolveShortcut(path)` - 完整解析快捷方式信息
  - `ResolvePathIfShortcut(path)` - 如果是快捷方式则解析，否则原样返回

## 多进程启动

### 配置方式

在添加或编辑游戏对话框的「启动进程」区域：

- **主启动进程（可选）**：主要的游戏启动程序路径
- **主进程启动参数（可选）**：主进程的命令行参数
- **第二启动进程（可选，最多2个）**：辅助程序路径（如 MOD 管理器、加速器等）
- **第二进程启动参数（可选）**：第二进程的命令行参数

### 启动流程

```
用户点击启动
  ↓
启动主进程 → 等待5秒检测是否存活
  ↓ (成功)          ↗ (5秒内退出)
启动第二进程       → 弹窗提示启动失败
  ↓ (成功)            终止监控周期
开始监听所有进程
  ↓
所有进程退出 → 执行存档备份
```

### 崩溃检测机制

- 每个进程启动后会等待 5 秒
- 如果进程在 5 秒内退出，判定为启动失败或崩溃
- 系统会：
  1. 立即终止已启动的其他进程
  2. 从托盘恢复主窗口显示
  3. 弹窗提示用户错误信息
  4. 提示内容示例：`"检测到 xxx 进程启动失败或崩溃（启动后 N 秒退出），本次监控周期将终止，请您排查后再次运行。"`

### 退出监测

- **单进程模式**：保持原有逻辑（包括 Steam stub 检测等）
- **多进程模式**：并行监听所有进程，所有进程全部退出后才触发存档备份

### 停止游戏

- 停止游戏时会尝试结束所有已启动的进程
- 通过 PID 和进程名两种方式确保所有关联进程被终止

## 数据模型变更

`Game` 模型新增字段（会持久化到 JSON 配置中）：

| 字段                   | 类型     | 说明                   |
|------------------------|----------|------------------------|
| SecondaryProcessPath   | string?  | 第二启动进程路径       |
| SecondaryProcessArgs   | string?  | 第二启动进程参数       |

运行时属性（不持久化）：

| 属性               | 类型        | 说明                     |
|--------------------|-------------|--------------------------|
| RunningPids        | List\<int\> | 所有运行中的进程 PID     |
| DisplayRunningPid  | string      | 显示用的 PID 文本        |
| AllProcessEntries  | List        | 所有配置的进程入口       |
