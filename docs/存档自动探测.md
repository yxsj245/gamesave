# 存档自动探测功能

## 功能概述

当游戏已设置启动进程但未设置存档目录时，点击"启动游戏"会自动进入**存档目录探测模式**。该功能基于 ETW（Event Tracing for Windows）内核事件追踪技术，监控游戏进程的文件写入操作，自动识别疑似存档目录。

## 三区块探测

探测结果按来源分为三个独立区块展示，每个区块独立计分和排序：

| 区块 | 说明 | 优先级 |
|------|------|--------|
| 📁 用户目录 | AppData、Documents、Saved Games 等 C 盘用户目录 | 最高，排在最上方 |
| 🎮 游戏安装目录 | 游戏启动进程 exe 所在的目录及其子目录 | 第二 |
| 📂 其他目录 | 不属于上面两个区域的写入位置 | 第三 |

## 使用步骤

### 1. 添加游戏

添加游戏时，填写**启动进程路径**但**不填写存档目录**（留空）。

### 2. 启动探测

点击"启动游戏"按钮后，系统会弹出提示：

- 即将启动存档探测模式
- 探测期间 CPU 和内存将会短暂上升
- 需要管理员权限运行

点击"开始探测"后：

- **非管理员运行**：应用将退出并以管理员身份重新启动（触发 UAC 确认对话框）
- **已是管理员**：直接进入探测流程

### 3. 探测中

探测启动后：

1. 游戏将正常启动
2. 主窗口进入系统托盘
3. 屏幕上弹出一个**置顶的探测结果窗口**

在探测窗口中：

- 实时显示捕获到的疑似存档目录列表，按三个区块分组展示
- 每个目录显示**写入次数**和**概率得分**
- 点击目录前的 **▶ 箭头**可展开查看该目录下的具体文件
- 在目录前的**勾选框**中选择您确认的存档目录（可跨区块多选）

> **提示：** 在游戏中执行一次存档操作（如手动保存、自动存档触发等），探测窗口会立即捕获并显示疑似存档目录。

### 4. 确认选择

勾选正确的存档目录后，点击"确认选择"按钮：

- 探测服务自动停止
- 选中的目录将被设置为该游戏的存档路径
- 系统托盘显示操作结果通知

## 技术原理

- 使用 ETW 内核级事件追踪，与 Procmon 原理相同
- 监控 `FileIOWrite` 和 `FileIOCreate` 事件
- **用户目录区块**：关注 AppData、LocalLow、Documents、Saved Games、Desktop
- **游戏目录区块**：关注游戏启动进程 exe 所在目录及其子目录
- **其他目录区块**：关注排除系统路径后的其他写入位置
- 自动排除缓存、日志、浏览器、Windows 系统目录、Program Files 等无关目录
- 按写入频率和子目录数量计算"概率得分"
- 目录名包含 save/saves/savegame/savedata 关键字时自动提升得分至 99

## 注意事项

- **管理员权限**：ETW 内核追踪必须以管理员身份运行
- **性能影响**：探测期间 ETW 会增加少量 CPU 和内存开销，停止探测后恢复正常
- **兼容性**：Windows 10 及以上版本
