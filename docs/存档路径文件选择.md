# 存档路径支持选择文件

> 存档路径现在同时支持选择「文件夹」和「单独文件」，并内置冲突校验机制。

---

## 功能说明

### 选择方式

在添加游戏、编辑游戏、导入游戏的存档路径区域，现在有两个浏览按钮：

| 按钮 | 功能 | 适用场景 |
|------|------|---------|
| **文件夹** | 选择整个目录，备份该目录下所有文件 | 大多数游戏存档（如 `%APPDATA%\GameName`） |
| **文件** | 选择单个文件 | 存档只有一个文件的游戏（如某个 `.sav` 或 `.dat` 文件） |

当然，仍然可以直接在文本框中手动输入路径。

### 冲突校验

为防止备份时出现重复/冲突，系统会进行以下校验：

1. **选择文件时**：检查该文件是否位于已添加的文件夹内
   - ❌ 已添加 `C:\Games\SaveData`，再选择 `C:\Games\SaveData\save.dat` → 冲突
   - ✅ 已添加 `C:\Games\SaveData`，再选择 `D:\OtherGame\save.dat` → 通过

2. **选择文件夹时**：检查该文件夹是否包含已添加的文件
   - ❌ 已添加 `C:\Games\SaveData\save.dat`，再选择 `C:\Games\SaveData` → 冲突
   - ✅ 已添加 `C:\OtherPath\save.dat`，再选择 `C:\Games\SaveData` → 通过

冲突时会弹窗提示，阻止添加冲突路径。

---

## 备份与还原行为

### 备份（打包）

- **文件夹路径**：递归打包文件夹下的所有文件（原有行为不变）
- **文件路径**：将该单个文件打包进 `.tar` 存档

### 还原（解压）

- **文件夹路径**：清空文件夹后解压存档到该目录（原有行为不变）
- **文件路径**：删除旧文件后解压存档到文件所在目录

### 路径类型判断逻辑

系统通过以下方式判断路径是文件还是目录：

1. `Directory.Exists(path)` 返回 `true` → 目录
2. `File.Exists(path)` 返回 `true` 且不是目录 → 文件
3. 路径不存在时，通过 `Path.HasExtension()` 辅助判断
