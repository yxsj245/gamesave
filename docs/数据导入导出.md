# 数据导入导出功能说明

> 导入导出功能位于 **设置** 页面的 **数据管理** 卡片中。

---

## 导出游戏

将游戏配置和存档备份文件打包为 `.zip` 文件，方便备份到其他位置或迁移到其他设备。
自定义图标文件也会一起导出，导入后可自动恢复到对应游戏。

### 操作步骤

1. 进入 **设置** 页面
2. 在 **数据管理** 区域点击 **「导出游戏」** 按钮
3. 弹出游戏选择窗口，默认全选所有游戏
   - 勾选/取消勾选 **「全选」** 可快速选中或取消全部游戏
   - 也可单独勾选需要导出的游戏
4. 点击 **「导出」**，选择保存路径和文件名
5. 等待导出完成，弹出结果提示

### 导出文件结构

```
GameSave_Export_20260310_120000.zip
├── manifest.json               # 导出清单（版本、时间、游戏概要）
├── <游戏ID-1>/
│   ├── game.json               # 游戏配置（名称、存档路径等）
│   └── saves/
│       ├── 12345_退出存档.tar   # 存档备份文件
│       └── 67890_手动存档.tar
│   └── icon/                   # 自定义图标文件（如有）
├── <游戏ID-2>/
│   ├── game.json
│   └── saves/
│       └── ...
```

> 💡 游戏存档路径（`SaveFolderPath`）以环境变量形式保存（如 `%APPDATA%\Game`），在不同设备导入时会自动适配路径。

---

## 导入数据

从导出的 `.zip` 文件中恢复游戏配置和存档。

### 操作步骤

1. 进入 **设置** 页面
2. 在 **数据管理** 区域点击 **「导入数据」** 按钮
3. 选择之前导出的 `.zip` 文件
4. 弹出预览窗口，显示文件中包含的游戏列表：
   - 游戏名称
   - 存档数量
   - 存档路径
   - 是否已有同名游戏（⚠️ 标注）
5. 确认后点击 **「开始导入」**
6. 导入完成后弹出结果摘要

### 注意事项

- **重复游戏自动跳过**：如果本地已有同名或同 ID 的游戏，导入时会自动跳过并在结果中提示
- **路径自动适配**：存档路径中的环境变量（如 `%APPDATA%`、`%USERPROFILE%`）会在当前设备上自动展开为正确的绝对路径
- **自动兼容存档目录结构**：如果导出包中的存档位于 `saves/` 子目录下，导入时会自动识别并还原到本地可识别的位置，无需手动移动 `.tar` 文件
- **图标一并迁移**：如果导出包中包含自定义图标，导入后会一起写回游戏目录并显示在列表中
- **导入后刷新**：导入完成后需返回主页刷新游戏列表以查看新导入的游戏
